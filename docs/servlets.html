📌 1. What is a Servlet?

A Servlet is a Java program that runs on a web server and handles client requests.

👉 It is mainly used to:

  • Process HTTP requests
  • Generate dynamic web content
  • Interact with databases
  • Act as a controller in MVC architecture

Definition: A Servlet is a server-side Java class that processes requests and generates responses dynamically.

📌 2. Why Servlets?

Before Servlets:

  • CGI (Common Gateway Interface)
  • Each request created a new process → slow & heavy

After Servlets:

  • Runs inside JVM
  • Uses threads → fast & scalable

Advantages

  • High performance (multithreading)
  • Platform independent
  • Secure
  • Reusable
  • Integrated with Java ecosystem

📌 3. Servlet Architecture

Client (Browser) sends request
Web Server forwards request to Servlet Container
Servlet Container:
    Creates request & response objects
    Calls servlet methods
Servlet processes request
Response sent back to client

Components

  • Client (Browser)
  • Web Server
  • Servlet Container (Apache Tomcat)
  • Servlet

📌 4. Servlet Life Cycle

init()

public void init() {
    System.out.println("Servlet Initialized");
}

service()

public void service(ServletRequest req, ServletResponse res) {
    System.out.println("Request processed");
}

destroy()

public void destroy() {
    System.out.println("Servlet Destroyed");
}

📌 5. Types of Servlets

Generic Servlet

public class MyServlet extends GenericServlet {
    public void service(ServletRequest req, ServletResponse res) {
        System.out.println("Generic Servlet");
    }
}

HTTP Servlet

public class MyServlet extends HttpServlet {

    protected void doGet(HttpServletRequest req, HttpServletResponse res) {
        System.out.println("GET Request");
    }

    protected void doPost(HttpServletRequest req, HttpServletResponse res) {
        System.out.println("POST Request");
    }
}
