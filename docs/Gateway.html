Spring Boot Gateway Notes

What is API Gateway?

An API Gateway is the single entry point for all client requests in a microservices architecture.

Client → API Gateway → Microservices

Gateway Handles

  • Routing
  • Authentication
  • Authorization
  • Load Balancing
  • Logging
  • Rate Limiting
  • Monitoring
  • Security

Spring Cloud Gateway

Spring Boot provides gateway support using Spring Cloud Gateway.

Built On

  • Spring Boot
  • Spring WebFlux
  • Reactive Programming

Why Use Gateway?

Without Gateway

Client → Account Service Client → User Service Client → Payment Service

Problems

  • Multiple URLs exposed
  • Security difficult
  • CORS issues
  • Hard to monitor

With Gateway

Client → Gateway → Services

Benefits

  • Centralized security
  • Single endpoint
  • Easy routing
  • Better monitoring
  • Load balancing
  • Dynamic service discovery

<dependency>

Basic Configuration

server.port=8080

spring.application.name=API-GATEWAY

Static Routing

spring.cloud.gateway.routes[0].id=account-service
spring.cloud.gateway.routes[0].uri=http://localhost:8081
spring.cloud.gateway.routes[0].predicates[0]=Path=/accounts/**

Meaning

Property Meaning
id Route name
uri Target service
predicates Matching condition

Flow

Request: http://localhost:8080/accounts/1 Gateway checks: Path=/accounts/** Forward to: http://localhost:8081/accounts/1

Dynamic Routing Using Eureka

spring.cloud.gateway.discovery.locator.enabled=true
spring.cloud.gateway.discovery.locator.lower-case-service-id=true

Example

If Account Service runs on: http://localhost:8081 Then access through gateway: http://localhost:8080/account-service/accounts/1

Using Load Balancer

spring.cloud.gateway.routes[0].id=account-service
spring.cloud.gateway.routes[0].uri=lb://ACCOUNT-SERVICE
spring.cloud.gateway.routes[0].predicates[0]=Path=/accounts/**

Meaning of lb://

lb:// means Load Balance using Eureka.

Multiple Instances Example

ACCOUNT-SERVICE localhost:8081 localhost:8082 localhost:8083

Common Predicates

Predicate Description
Path Matches URL paths
Method Matches HTTP methods
Header Checks headers
Query Checks query parameters

Filters

AddRequestHeader=X-Request-Red, Blue
RemoveRequestHeader=Cookie
RewritePath=/accounts/(?<segment>.*), /${segment}

Gateway with Eureka Architecture

Eureka Server ↑ | ------------------------------ | | | Account Service User Service Payment Service ↑ API Gateway ↑ Client

Important Ports Example

Service Port
Eureka Server 8761
Gateway 8080
Account Service 8081
User Service 8082

Gateway Flow Internally

  1. Client sends request
  2. Gateway checks predicates
  3. Route matched
  4. Filters applied
  5. Request forwarded
  6. Response returned

Advantages of Gateway

  • Centralized routing
  • Security
  • Logging
  • JWT authentication
  • Load balancing
  • Service discovery
  • Monitoring
  • Rate limiting

Disadvantages

  • Single point of failure
  • Adds complexity
  • Reactive programming learning curve
