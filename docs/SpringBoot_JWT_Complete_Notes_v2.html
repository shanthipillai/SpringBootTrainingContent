Building JWT Token

public String generateToken(String username){
 return Jwts.builder()
     .subject(username)
     .issuedAt(new Date())
     .expiration(new Date(System.currentTimeMillis()+3600000))
     .signWith(getSigningKey())
     .compact();
}

Adding Custom Claims

.claim("role","ADMIN")
.claim("department","IT")

Parsing JWT Token

public String extractUsername(String token){
 return Jwts.parser()
     .verifyWith(getSigningKey())
     .build()
     .parseSignedClaims(token)
     .getPayload()
     .getSubject();
}

JWT Validation

public boolean validateToken(String token,String username){
 return extractUsername(token).equals(username)
        && !isTokenExpired(token);
}

JWT vs Session Management

Session Management

Server creates and stores user session information. Client sends only Session ID.

User Login
    |
Server Creates Session
    |
Session ID Generated
    |
Client Stores Session ID
    |
Every Request Sends Session ID

JWT Authentication

Server generates a signed token. No user session is stored on the server.

User Login
    |
Generate JWT
    |
Client Stores JWT
    |
Authorization Header
    |
Server Validates JWT

Why JWT is Stateless?

{
 "sub":"shanthi",
 "role":"ADMIN",
 "exp":1750000000
}

The server validates the token signature and does not maintain session state.

Session Example

HttpSession session = request.getSession();
session.setAttribute("username","shanthi");

JWT Example

String token = jwtService.generateToken("shanthi");
jwtService.validateToken(token, username);
