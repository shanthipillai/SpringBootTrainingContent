Servlet & JSP Complete Notes

JSP Directives

What are JSP Directives?

JSP Directives provide instructions to the JSP container during translation phase (JSP → Servlet).

Syntax

<%@ directive attribute="value" %>

Types of JSP Directives

1. Page Directive

Purpose

Defines page-level settings for a JSP.

<%@ page attribute="value" %>

Common Attributes

<%@ page import="java.util.*, java.sql.*" %>
<%@ page contentType="text/html;charset=UTF-8" %>
<%@ page pageEncoding="UTF-8" %>
<%@ page session="true" %>
<%@ page isELIgnored="false" %>
<%@ page errorPage="error.jsp" %>
<%@ page isErrorPage="true" %>

Example

<%@ page import="java.util.*" contentType="text/html;charset=UTF-8" %>
  • Applies to entire JSP
  • Translated into servlet code
  • Usually placed at top

2. Include Directive

Purpose

Includes another file at compile time.

<%@ include file="header.jsp" %>
  • Static include
  • Code copied into servlet
  • Faster execution
  • Same request

Example

<%@ include file="header.jsp" %>
<h1>Welcome</h1>
<%@ include file="footer.jsp" %>
Include Directive jsp:include
Compile-time Runtime
Static Dynamic
Faster Slightly slower

3. Taglib Directive

Purpose

Used to include JSTL or custom tag libraries.

<%@ taglib uri="http://java.sun.com/jsp/jstl/core" prefix="c" %>

Example

<c:forEach var="user" items="${users}">
    ${user.name}
</c:forEach>
  • Avoid Java code
  • Clean and readable
  • Works with EL

Directive vs Action vs Scriptlet

Feature Directive Action Tag Scriptlet
Syntax <%@ %> <jsp:...> <% %>
Time Compile Runtime Runtime
Purpose Config Dynamic tasks Java code

Final Summary

  • JSP Directives work at translation phase
  • Types: page, include, taglib
  • <%@ include %> = compile-time
  • <jsp:include> = runtime
