JUnit 5 Testing Complete Notes

What is JUnit?

JUnit is the most popular testing framework for Java applications. It is used to perform unit testing and verify whether the application behavior matches the expected output.

Uses of JUnit

  • Test individual methods and classes
  • Verify expected output
  • Prevent bugs during development
  • Support Test Driven Development (TDD)
  • Improve application quality
  • Enable automated testing

JUnit 5 Modules

  • JUnit Platform
  • JUnit Jupiter
  • JUnit Vintage

Why Do We Need Unit Testing?

Without Testing


public int add(int a, int b) {
    return a + b;
}

If someone accidentally changes the logic:


public int add(int a, int b) {
    return a - b;
}

The application may fail and produce incorrect results. Unit tests immediately detect such changes.

Basic JUnit Test

Calculator Class


public class Calculator {

    public int add(int a, int b) {
        return a + b;
    }
}

Test Class


import static org.junit.jupiter.api.Assertions.*;
import org.junit.jupiter.api.Test;

public class CalculatorTest {

    @Test
    void testAdd() {

        Calculator calculator = new Calculator();

        int result = calculator.add(10, 20);

        assertEquals(30, result);
    }
}

Common JUnit Annotations

Annotation Purpose
@Test Marks a test method
@BeforeEach Runs before every test
@AfterEach Runs after every test
@BeforeAll Runs once before all tests
@AfterAll Runs once after all tests
@Disabled Skips a test
@DisplayName Custom test name
@ParameterizedTest Runs same test with multiple inputs

Mockito Integration

Service Class


@Service
public class EmployeeService {

    @Autowired
    EmployeeRepository repository;

    public List<Employee> getAllEmployees() {
        return repository.findAll();
    }
}

Test Class


@ExtendWith(MockitoExtension.class)
class EmployeeServiceTest {

    @Mock
    EmployeeRepository repository;

    @InjectMocks
    EmployeeService service;

    @Test
    void testGetAllEmployees() {

        List<Employee> employees =
            List.of(
                new Employee(1,"John")
            );

        when(repository.findAll())
                .thenReturn(employees);

        List<Employee> result =
                service.getAllEmployees();

        assertEquals(1, result.size());

        verify(repository, times(1))
                .findAll();
    }
}

Mock vs InjectMocks

Annotation Purpose
@Mock Creates fake object
@InjectMocks Injects mocks into class under test
@Spy Partial mock
@Captor Captures arguments

Verify Methods


verify(repository).findAll();

verify(repository, times(1))
        .findAll();

verify(repository, never())
        .deleteById(1);
