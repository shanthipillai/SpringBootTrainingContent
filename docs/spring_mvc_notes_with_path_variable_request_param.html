📦 PathVariable vs RequestParam (Detailed)

🔹 1. Using Path Variables (@PathVariable)

👉 Used when value is part of the URL path

Example URL: http://localhost:8080/user/101

@GetMapping("/user/{id}") public String getUser(@PathVariable int id) { return "User ID: " + id; }
  • {id} → placeholder in URL
  • @PathVariable → binds value from URL

🔹 Multiple Path Variables

URL: http://localhost:8080/user/101/John

@GetMapping("/user/{id}/{name}") public String getUser(@PathVariable int id, @PathVariable String name) { return id + " - " + name; }

🔹 Custom Name Mapping

@GetMapping("/user/{id}") public String getUser(@PathVariable("id") int userId) { return "User ID: " + userId; }

🔹 2. Using Request Parameters (@RequestParam)

👉 Used when value is sent as query parameter

Example URL: http://localhost:8080/user?id=101

@GetMapping("/user") public String getUser(@RequestParam int id) { return "User ID: " + id; }

🔹 Multiple Request Params

URL: http://localhost:8080/user?id=101&name=John

@GetMapping("/user") public String getUser(@RequestParam int id, @RequestParam String name) { return id + " - " + name; }

🔹 Optional Parameters

@GetMapping("/user") public String getUser(@RequestParam(required = false) String name) { return "Name: " + name; }

🔹 Default Value

@GetMapping("/user") public String getUser(@RequestParam(defaultValue = "Guest") String name) { return name; }

🔥 PathVariable vs RequestParam

Feature PathVariable RequestParam
URL Style /user/101 /user?id=101
Usage Mandatory Optional
Use Case Resource identification Search / filter
