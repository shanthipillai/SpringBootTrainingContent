Ngrok Complete Notes

Introduction

ngrok is a secure tunneling tool that exposes your local application to the internet using a public URL.

It helps developers test webhooks, APIs, mobile backends, and remote demos without deploying applications to production servers.

What is Ngrok?

ngrok creates a secure public URL for your local application.

Example:

Your Spring Boot application runs locally:

http://localhost:8080

ngrok converts it into:

https://abcd1234.ngrok-free.app
Anyone on the internet can now access your local application securely.

Real-Time Example

Suppose your Spring Boot application is running locally: 

http://localhost:8080

Normally only your system can access localhost.

External services like GitHub cannot access localhost directly.

Using ngrok

ngrok http 8080

ngrok generates:

https://abc123.ngrok-free.app
Now GitHub, browsers, Postman, and external systems can access your local application.

Installation Steps (Windows)

Step 1: Download Ngrok

Open:

https://ngrok.com/download

Download Windows ZIP file.

Step 2: Extract ZIP

Extract ZIP to:

C:\ngrok

You will see:

ngrok.exe

Step 3: Create Account

Open:

https://dashboard.ngrok.com/signup

Sign up using:

  • Google
  • GitHub
  • Email

Step 4: Get Authentication Token

Open dashboard:

https://dashboard.ngrok.com/get-started/your-authtoken

You will see command:

ngrok config add-authtoken YOUR_TOKEN

Step 5: Configure Token

Open Command Prompt inside ngrok folder and run:

ngrok config add-authtoken YOUR_TOKEN

Example:

ngrok config add-authtoken 2abxxxxxxxxxxxxx
Success message:

Authtoken saved

Running Ngrok

Start Spring Boot Application

localhost:8080

Run ngrok

ngrok http 8080

Output

Forwarding https://abcd1234.ngrok-free.app -> http://localhost:8080
Your local application is now publicly accessible.

Testing in Browser

Important Note

Free ngrok URLs change every restart.

Example:

Today: 
https://abc.ngrok-free.app
Tomorrow: 
https://xyz.ngrok-free.app
Update webhook URLs after restart.

Common Errors

1. ERR_NGROK_8012

Meaning:

Application is not running on localhost.

Solution:

  • Start Spring Boot application
  • Verify correct port number

2. 502 Bad Gateway

Meaning:

ngrok connected but backend application failed.

Possible Reasons:

  • Application stopped
  • Wrong port
  • Endpoint error
  • Controller exception

3. Authentication Failed

Reason:

Authtoken not configured.

Solution:

ngrok config add-authtoken YOUR_TOKEN
