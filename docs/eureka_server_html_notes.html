📘 Eureka Server (Service Registry) – Complete Training Notes

1. Introduction

Eureka Server is a service registry used in microservices architecture. It is part of Spring Cloud Netflix Eureka and helps services register themselves and discover other services dynamically.

Instead of hardcoding URLs, services communicate using service names.

2. Why Needed (Real-world)

Problem (Without Eureka)

  • Services must know IP address
  • Services must know port
  • If service restarts → URL changes → communication breaks

Solution (With Eureka)

  • Services register with Eureka
  • Other services ask Eureka for location
  • Dynamic discovery = no hardcoding

3. Core Concepts

  • Service Registry
  • Service Discovery
  • Client-side Load Balancing
  • Heartbeat Mechanism

4. Architecture

Components

  • Eureka Server – Central registry
  • Eureka Client – Microservices

5. How It Works (Lifecycle)

  1. Service starts
  2. Registers with Eureka
  3. Sends heartbeat (renewal)
  4. Other services query Eureka
  5. Get service instance details

Example:

ACCOUNT-SERVICE → registered
PAYMENT-SERVICE → queries Eureka → gets ACCOUNT-SERVICE URL

6. Key Features

  • Service Registration
  • Service Discovery
  • Load Balancing (client-side)
  • Fault Tolerance
  • Health Monitoring

7. Important Terms

  • Instance: One running service
  • Registry: List of services
  • Heartbeat: Sent every 30 seconds
  • Lease Renewal: Confirms service is alive
  • Eviction: Removes dead services

9. Creating Eureka Server

<dependency> <groupId>org.springframework.cloud</groupId> <artifactId>spring-cloud-starter-netflix-eureka-server</artifactId> </dependency> @EnableEurekaServer @SpringBootApplication public class EurekaServerApplication { public static void main(String[] args) { SpringApplication.run(EurekaServerApplication.class, args); } } server: port: 8761 eureka: client: register-with-eureka: false fetch-registry: false

10. Creating Eureka Client

<dependency> <groupId>org.springframework.cloud</groupId> <artifactId>spring-cloud-starter-netflix-eureka-client</artifactId> </dependency> spring: application: name: ACCOUNT-SERVICE eureka: client: service-url: defaultZone: http://localhost:8761/eureka @EnableDiscoveryClient

13. Common Configurations

eureka: instance: hostname: localhost prefer-ip-address: true client: register-with-eureka: true fetch-registry: true

14. Advantages

  • No hardcoded URLs
  • Auto scaling support
  • Dynamic service discovery
  • Fault tolerant

15. Disadvantages

  • Extra component to maintain
  • Slight latency
  • Replaced by Kubernetes DNS in modern setups
