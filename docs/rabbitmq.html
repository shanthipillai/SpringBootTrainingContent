RabbitMQ Installation Guide for Windows

Introduction

RabbitMQ is an open-source message broker software that enables applications, systems, and services to communicate with each other using messages. It supports multiple messaging protocols and is widely used in distributed systems, microservices architecture, asynchronous processing, and event-driven applications.

Why RabbitMQ is Needed

  • Decouples microservices communication
  • Supports asynchronous message processing
  • Improves application scalability
  • Provides reliable message delivery
  • Useful for background jobs and event-driven systems

Prerequisites

  • Windows 10 / Windows 11
  • Administrator privileges
  • Internet connection

Step 1: Install Erlang OTP

What is Erlang?

RabbitMQ is built on top of Erlang OTP runtime. Therefore, Erlang must be installed before installing RabbitMQ.

Erlang Download Links

  • Erlang Official Downloads Page:
    https://www.erlang.org/downloads
  • Direct Erlang Windows 64-bit Installer:
    https://github.com/erlang/otp/releases
  • Erlang Official Website:
    https://www.erlang.org/

Installation Steps

  1. Open Erlang download page.
  2. Download latest Windows 64-bit installer (.exe).
  3. Run installer as Administrator.
  4. Keep default installation location.
  5. Complete installation.

Verify Erlang Installation

Open Command Prompt and execute:

erl
If installation is successful, Erlang shell will open.

Step 3: Enable RabbitMQ Management Plugin

The management plugin provides a browser-based dashboard for monitoring and managing RabbitMQ.

Command

rabbitmq-plugins enable rabbitmq_management

Expected Output

The following plugins have been enabled:

rabbitmq_management
rabbitmq_management_agent
rabbitmq_web_dispatch

Step 4: Restart RabbitMQ Service

Using Command Prompt

net stop RabbitMQ
net start RabbitMQ

Using Windows Services

  1. Press Windows + R
  2. Type services.msc
  3. Locate RabbitMQ service
  4. Restart service manually

Step 5: Open RabbitMQ Dashboard

Open browser:

http://localhost:15672

Default Login Credentials

Username Password
guest guest

Important Ports

Port Purpose
5672 RabbitMQ Server Communication
15672 Management Dashboard

Verify RabbitMQ Service

Execute:

rabbitmqctl status
RabbitMQ server details and status information will be displayed.

Common Issues

RabbitMQ not detecting Erlang

Configure environment variables manually.

Set ERLANG_HOME

C:\Program Files\Erlang OTP\erl-28.0

Add to PATH

%ERLANG_HOME%\bin
Restart Command Prompt after updating environment variables.

Best Practices

  • Always install compatible Erlang version for RabbitMQ.
  • Enable management plugin for monitoring.
  • Use strong credentials in production.
  • Monitor queues regularly.
  • Configure persistence for critical messages.

Common Mistakes

  • Installing incompatible Erlang version.
  • Forgetting to enable management plugin.
  • Not restarting RabbitMQ service after plugin installation.
  • Incorrect environment variable configuration.
