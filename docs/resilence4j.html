Resilience4j – Circuit Breaker Notes

What is Resilience4j?

Resilience4j is a lightweight fault tolerance library used in Spring Boot microservices to make applications more resilient and stable during failures.

Features

  • Circuit Breaker
  • Retry
  • Bulkhead
  • Rate Limiter
  • Time Limiter

It is the modern replacement for Netflix Hystrix.

What is Circuit Breaker?

A Circuit Breaker prevents repeated calls to a failing service.

Example Scenario

  • Loan Service calls Payment Service
  • Payment Service goes down
Without Circuit Breaker → Loan Service keeps calling the failed service repeatedly.

With Circuit Breaker → Calls are stopped temporarily after repeated failures.

Real-Time Flow

Client
   |
   v
Loan Service
   |
Circuit Breaker
   |
Payment Service

Circuit Breaker States

1. CLOSED State

  • Normal state
  • All requests are allowed
  • Failures are monitored

2. OPEN State

  • Requests are blocked
  • No call goes to downstream service
  • Fails immediately

3. HALF_OPEN State

  • Allows limited test requests
  • Checks whether service recovered

Circuit Breaker State Diagram

Important Properties

Property Meaning
failureRateThreshold Failure percentage to open circuit
slidingWindowSize Number of requests monitored
waitDurationInOpenState Time circuit stays OPEN
permittedNumberOfCallsInHalfOpenState Test calls allowed in HALF_OPEN
minimumNumberOfCalls Minimum requests before calculating failure rate

Maven Dependency

<dependency>
    <groupId>io.github.resilience4j</groupId>
    <artifactId>resilience4j-spring-boot3</artifactId>
</dependency>

application.yml Configuration

resilience4j:
  circuitbreaker:
    instances:
      paymentService:
        registerHealthIndicator: true
        slidingWindowSize: 10
        minimumNumberOfCalls: 5
        failureRateThreshold: 50
        waitDurationInOpenState: 10s
        permittedNumberOfCallsInHalfOpenState: 3
        automaticTransitionFromOpenToHalfOpenEnabled: true

Service Example

@Service
public class PaymentService {

    @CircuitBreaker(
        name = "paymentService",
        fallbackMethod = "fallbackMethod"
    )
    public String makePayment() {

        RestTemplate restTemplate = new RestTemplate();

        return restTemplate.getForObject(
                "http://localhost:8082/payment",
                String.class);
    }

    public String fallbackMethod(Exception ex) {
        return "Payment Service is temporarily unavailable";
    }
}

How It Works

Suppose:

  • slidingWindowSize = 10
  • failureRateThreshold = 50

If 5 out of 10 requests fail, then Circuit Breaker becomes OPEN.

Benefits of Circuit Breaker

  • Prevents cascading failures
  • Improves response time
  • Avoids unnecessary remote calls
  • Helps services recover
  • Improves system stability

Common Real-Time Use Cases

Service Why Circuit Breaker?
Payment Gateway Third-party failures
Notification Service Avoid blocking main flow
Inventory Service Prevent cascading failures
External APIs Timeout handling
Banking APIs Fast failure response

Retry vs Circuit Breaker

Retry Circuit Breaker
Reattempts requests Stops requests
Handles temporary failures Prevents overload
