Servlet & JSP Concepts Notes

1. RequestDispatcher

What is it?

RequestDispatcher is used to forward or include resources (JSP/Servlet) within the server.

RequestDispatcher rd = request.getRequestDispatcher("home.jsp");

Types of Operations

1. Forward
rd.forward(request, response);

2. Include
rd.include(request, response);

2. Request Forward

Definition

Transfers request from one servlet/JSP to another resource.

request.setAttribute("name", "Shanthi");
RequestDispatcher rd = request.getRequestDispatcher("home.jsp");
rd.forward(request, response);

Key Points

  • Same request & response
  • URL does NOT change
  • Control does NOT return back
  • Request attributes are available

Use Case

  • Login → Dashboard
  • Controller → JSP

3. Request Include

Definition

Includes output of another resource into current response.

RequestDispatcher rd = request.getRequestDispatcher("header.jsp");
rd.include(request, response);

Key Points

  • Same request
  • Output is appended
  • Control returns back
  • Used for reusable components

Use Case

  • Header / Footer / Navbar

4. sendRedirect

Definition

Sends response to browser → browser makes new request

response.sendRedirect("home.jsp");

Key Points

  • New request
  • URL changes
  • Request data lost
  • Slower than forward

Forward vs Redirect

Feature Forward Redirect
Request Same New
URL change No Yes
Speed Fast Slower
Request attributes Available Lost

5. HttpSession

Definition

Used to store user data across multiple requests.

HttpSession session = request.getSession();
session.setAttribute("user", "Shanthi");

Session Methods

Validate (Concept)

HttpSession session = request.getSession(false);
if(session != null) {
    // valid session
}

Invalidate

session.invalidate();

Effect

  • Session destroyed
  • All data removed
  • User logged out

Session Timeout

<session-config>
    <session-timeout>30</session-timeout>
</session-config>

session.setMaxInactiveInterval(1800);

6. Cookie

Definition

Small data stored in browser.

Cookie cookie = new Cookie("user", "Shanthi");
response.addCookie(cookie);

Read Cookie

Cookie[] cookies = request.getCookies();

Key Points

  • Stored in browser
  • Used for login tracking
  • Smaller size than session
  • Can be disabled

8. JSP Scriptlets

Old Style (Avoid)

<%
String name = "Shanthi";
out.println(name);
%>

Issues

  • Mixing Java + HTML
  • Hard to maintain

Preferred

${name}

9. JSTL (JSP Tag Libraries)

Definition

Standard tags to avoid Java code in JSP.

<dependency>
    <groupId>javax.servlet</groupId>
    <artifactId>jstl</artifactId>
</dependency>

<%@ taglib uri="http://java.sun.com/jsp/jstl/core" prefix="c" %>

Common Tags

if

<c:if test="${user != null}">
    Welcome User
</c:if>

forEach

<c:forEach var="u" items="${users}">
    ${u.name}
</c:forEach>

choose

<c:choose>
    <c:when test="${user == null}">
        Login
    </c:when>
    <c:otherwise>
        Welcome
    </c:otherwise>
</c:choose>
