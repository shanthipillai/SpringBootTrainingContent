Spring Boot Feign Client - Complete Notes

What is Feign Client?

Feign Client is a declarative HTTP client in Spring Boot.

Instead of writing RestTemplate or WebClient code, you define an interface and Spring generates the implementation.

Dependency

<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-openfeign</artifactId>
</dependency>

Enable Feign

@SpringBootApplication
@EnableFeignClients
public class AccountServiceApplication {
}

Basic Feign Client

@FeignClient(name = "card-service", url = "http://localhost:8081")
public interface CardClient {

    @GetMapping("/account/{accountId}")
    List<Card> getCards(@PathVariable Long accountId);
}

How It Works

  • Create interface
  • Add annotations
  • Spring creates proxy implementation
  • Internally uses HTTP calls

Service Layer Usage

@Service
public class AccountService {

    @Autowired
    private CardClient cardClient;

    public List<Card> fetchCards(Long accountId) {
        return cardClient.getCards(accountId);
    }
}

Key Annotations

AnnotationPurpose
@FeignClientDefines external service
@GetMappingHTTP GET call
@PostMappingHTTP POST call
@PathVariablePath param
@RequestParamQuery param
@RequestBodySend body

Using Service Name (Eureka)

@FeignClient(name = "CARD-SERVICE")

Requires service registered in Eureka and load balancing enabled.

Without URL (Recommended)

@FeignClient(name = "CARD-SERVICE")
  • Uses service discovery
  • No hardcoding

Request with Body

@PostMapping("/cards")
Card createCard(@RequestBody Card card);

Multiple Parameters

@GetMapping("/cards")
List<Card> getCards(
    @RequestParam String type,
    @RequestParam String status
);

Exception Handling (Fallback)

@FeignClient(name = "CARD-SERVICE", fallback = CardFallback.class)
@Component
public class CardFallback implements CardClient {

    @Override
    public List<Card> getCards(Long accountId) {
        return new ArrayList<>();
    }
}

Timeout Configuration

feign:
  client:
    config:
      default:
        connectTimeout: 5000
        readTimeout: 5000

Logging

logging:
  level:
    com.example.client: DEBUG

Feign vs RestTemplate vs WebClient

FeatureFeignRestTemplateWebClient
StyleDeclarativeImperativeReactive
CodeLessMoreMedium
BlockingYesYesNo
Best ForMicroservicesLegacyReactive apps

Advantages

  • Less boilerplate
  • Easy integration
  • Clean code
  • Service discovery support
  • Built-in load balancing

Disadvantages

  • Blocking (not reactive)
  • Less control than WebClient
  • Needs fallback handling

When to Use

  • Microservices communication
  • Simple REST calls
  • When using Eureka

When NOT to Use

  • Reactive applications → use WebClient
  • High-performance async flows

Important Notes

  • Feign is blocking
  • Uses Spring MVC annotations
  • Works best with Spring Cloud
  • Supports fallback (Resilience)
  • Internally uses HTTP client (Ribbon/LoadBalancer)
