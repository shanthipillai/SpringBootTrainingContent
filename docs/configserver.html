Spring Cloud Config Server Complete Notes

1. Introduction

Spring Cloud Config Server is used to externalize and centralize configuration management for microservices.

Instead of storing configuration files separately inside every microservice, configurations are stored in a centralized repository such as:

  • GitHub Repository
  • Local File System
  • Native Storage

All microservices fetch configurations dynamically from the Config Server.

2. Why Config Server is Needed

Problem in Microservices

  • Many services exist
  • Configurations repeat across services
  • Updating configs manually is difficult
  • Environment-specific configs become hard to manage

Advantages of Config Server

  • Centralized configuration management
  • Dynamic refresh support
  • Environment-specific configuration
  • Easy maintenance
  • Better scalability
  • Secure property management

3. Config Server Architecture

Client Microservices | v Config Server | v GitHub / Local Config Repository

Detailed Flow

  1. Config Server starts
  2. Connects to Git repository
  3. Microservice starts
  4. Reads bootstrap.properties
  5. Connects to Config Server
  6. Downloads configuration
  7. Application starts using downloaded properties

5. Enable Config Server

@SpringBootApplication
@EnableConfigServer
public class ConfigServerApplication {

    public static void main(String[] args) {
        SpringApplication.run(ConfigServerApplication.class, args);
    }
}

6. application.properties for Config Server

server.port=8888

spring.cloud.config.server.git.uri=https://github.com/username/config-repo
spring.cloud.config.server.git.clone-on-start=true

Explanation

Property Purpose
server.port Runs Config Server on port 8888
spring.cloud.config.server.git.uri GitHub repository URL
clone-on-start Clones repository during startup

11. Dynamic Refresh

Expose Refresh Endpoint

management.endpoints.web.exposure.include=refresh

Refresh API

POST /actuator/refresh

@RefreshScope Example

@RestController
@RefreshScope
public class TestController {

    @Value("${message}")
    private String message;

    @GetMapping("/msg")
    public String getMessage() {
        return message;
    }
}

13. Webhook Monitor Concept

Problem with Manual Refresh

Whenever GitHub configuration changes, developers must manually call: 

POST /actuator/refresh

Doing this manually for many services becomes difficult.

Solution

  • GitHub Webhook
  • Config Monitor Endpoint
  • Automatic Refresh

14. Monitor Endpoint

What is /monitor?

Config Server exposes: 

/monitor

This endpoint listens for GitHub webhook events.

Enable Monitor Endpoint

management.endpoints.web.exposure.include=*

Monitor API

POST http://localhost:8888/monitor

15. Ngrok Setup

Why Ngrok?

GitHub cannot access localhost directly.

Ngrok provides a public URL for local applications.

Start Config Server

http://localhost:8888

Run Ngrok

ngrok http 8888

Example Public URL

https://abc123.ngrok-free.app

16. GitHub Webhook Configuration

GitHub Steps

  1. Open GitHub Repository
  2. Go to Settings
  3. Select Webhooks
  4. Click Add Webhook

Webhook URL

https://abc123.ngrok-free.app/monitor

Content Type

application/json

Select Events

Just the push event

19. Difference Between Refresh and Monitor

/refresh /monitor
Manual API call Automatic webhook trigger
User triggers refresh GitHub triggers refresh
Actuator endpoint Webhook endpoint
Used locally Used in automation

20. Common Issues

502 Bad Gateway in Ngrok

  • Config Server not running
  • Wrong port configured
  • Application crashed
  • Ngrok connected to wrong port

Correct Command

ngrok http 8888

21. Best Practices

  • Store configs in separate Git repository
  • Never store passwords directly
  • Use profiles properly
  • Protect Config Server using security
  • Use actuator refresh carefully
  • Enable monitoring and logging

26. Interview Questions

What is Config Server?

Config Server provides centralized configuration management for microservices.

Why bootstrap.properties?

Because Config Server details are required before application startup.

Difference between bootstrap.properties and application.properties?

bootstrap.properties application.properties
Loads first Loads later
External config connection Business configuration
Config Server connection Application settings

What is @RefreshScope?

Reloads updated property values without restarting the application.

Why Ngrok?

GitHub cannot access localhost directly.

What does /monitor do?

Listens for webhook events and refreshes configuration automatically.

28. Final Summary

Spring Cloud Config Server centralizes configuration management for all microservices.

Using GitHub + Config Server + Webhook Monitor + Ngrok, applications can automatically refresh configurations without restarting services.

Config Server is one of the most important components in a Spring Boot Microservices Architecture.
