Spring WebClient Notes

What is WebClient?

WebClient is a non-blocking, reactive HTTP client introduced in Spring WebFlux.

It is used to:

  • Call external REST APIs
  • Consume microservices
  • Send GET, POST, PUT, DELETE requests
  • Work asynchronously using Mono and Flux

It replaces older:

  • RestTemplate (traditional synchronous client)

Dependency

Maven


<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-webflux</artifactId>
</dependency>

GET API Example

Service Class


@Service
public class ProductService {

    private final WebClient webClient;

    public ProductService(WebClient.Builder builder) {
        this.webClient = builder
                .baseUrl("https://fakestoreapi.com")
                .build();
    }

    public List<Product> getProducts() {

        Product[] products = webClient.get()
                .uri("/products")
                .retrieve()
                .bodyToMono(Product[].class)
                .block();

        return Arrays.asList(products);
    }
}

Controller


@RestController
@RequestMapping("/products")
public class ProductController {

    @Autowired
    private ProductService productService;

    @GetMapping
    public List<Product> getProducts() {
        return productService.getProducts();
    }
}

Mono vs Flux

Type Meaning
Mono 0 or 1 object
Flux Multiple objects

Examples:

  • Single User → Mono<User>
  • List of Products → Flux<Product>

POST Request Example


public Product addProduct(Product product) {

    return webClient.post()
            .uri("/products")
            .bodyValue(product)
            .retrieve()
            .bodyToMono(Product.class)
            .block();
}

PUT Request Example


public Product updateProduct(int id, Product product) {

    return webClient.put()
            .uri("/products/{id}", id)
            .bodyValue(product)
            .retrieve()
            .bodyToMono(Product.class)
            .block();
}

DELETE Request Example


public String deleteProduct(int id) {

    webClient.delete()
            .uri("/products/{id}", id)
            .retrieve()
            .bodyToMono(Void.class)
            .block();

    return "Deleted Successfully";
}

Important Methods

Method Purpose
get() HTTP GET
post() HTTP POST
put() HTTP PUT
delete() HTTP DELETE
uri() API endpoint
retrieve() Fetch response
bodyToMono() Convert response to Mono
bodyToFlux() Convert response to Flux
block() Converts async → sync

Difference Between RestTemplate and WebClient

RestTemplate WebClient

Error Handling


public Flux<Product> getProducts() {

    return webClient.get()
            .uri("/products")
            .retrieve()
            .onStatus(HttpStatusCode::is4xxClientError,
                    response -> Mono.error(new RuntimeException("Client Error")))
            .onStatus(HttpStatusCode::is5xxServerError,
                    response -> Mono.error(new RuntimeException("Server Error")))
            .bodyToFlux(Product.class);
}

Logging Request & Response


@Bean
public WebClient webClient() {

    return WebClient.builder()
            .filter(ExchangeFilterFunction.ofRequestProcessor(request -> {

                System.out.println("Request URL : " + request.url());

                return Mono.just(request);
            }))
            .build();
}

retrieve() vs exchangeToMono()

retrieve() exchangeToMono()
Simple response handling Full response control
Easy to use Advanced handling
Commonly used Used for custom logic

exchangeToMono Example


public Mono<String> getData() {

    return webClient.get()
            .uri("/products")
            .exchangeToMono(response -> {

                if (response.statusCode().is2xxSuccessful()) {
                    return response.bodyToMono(String.class);
                }

                return Mono.error(new RuntimeException("Failed"));
            });
}

Retry Example


public Flux<Product> getProducts() {

    return webClient.get()
            .uri("/products")
            .retrieve()
            .bodyToFlux(Product.class)
            .retry(3);
}

WebClient + Resilience4j


@CircuitBreaker(name = "product-service", fallbackMethod = "fallbackMethod")
public Flux<Product> getProducts() {

    return webClient.get()
            .uri("/products")
            .retrieve()
            .bodyToFlux(Product.class);
}

public Flux<Product> fallbackMethod(Exception ex) {

    return Flux.just(new Product(0, "Fallback Product"));
}

Common Interview Questions

1. Why WebClient over RestTemplate?

Because WebClient is non-blocking and reactive.

2. What is Mono?

A publisher that emits 0 or 1 item.

3. What is Flux?

A publisher that emits multiple items.

4. What is block()?

Converts async reactive flow into synchronous flow.

5. Is WebClient part of Spring MVC?

No. It belongs to Spring WebFlux.
